# SheNicest 政务工作台 UI 设计规则

本规则用于统一政策库、居民档案、智能匹配、居民画像和任务台账的视觉与交互。产品定位是基层工作人员使用的高信息密度业务工作台，不采用消费端卡片流或宣传型大留白设计。

## 1. 基础设计原则

1. 权威清晰：平台主视觉统一使用 `#F8DF70`，所有可见文字统一使用黑色。
2. 信息优先：减少装饰、插画、超大标题和无业务意义的留白。
3. 快速比较：需要比较多条、多字段数据时必须优先使用表格。
4. 结论可解释：匹配结论必须同时显示状态、依据和待核实信息。
5. 操作可追溯：重要页面显示数据来源、更新时间、责任人和人工核实状态。
6. 敏感信息最小展示：列表不展示完整身份证、电话和详细地址。

## 2. 页面框架

- 设计基准：1440 × 900 桌面端。
- 页面背景：`#F3F5F7`。
- 页面边距参考[北京市大兴区产业发展服务门户](https://chanye.bjdx.gov.cn/chanye/zcfw37/index.html)的实测布局：主体容器最大宽度1200px并水平居中；1280px浏览器视口下左右实际页边距约32px。
- 全站内容区统一使用 `width: min(1200px, calc(100% - 32px))`；窄屏左右安全间距为12px。
- 一级模块之间的垂直间距统一为20px；高密度工作台模块内部默认16px，简单提示区可使用10–12px，大型门户展示区最多30px。
- 顶部标题栏：`#F8DF70` 背景，深色文字，当前栏目使用深金色底边。
- 顶部右侧固定展示当前登录头像、登录状态和工作人员身份，身份信息保持一行且不抢占主导航。
- 列表与工作台页面不设置重复的宣传式标题区，进入页面后直接展示主要模块。
- 全局悬浮导航：固定在页面右侧，只提供政策库、居民档案和任务台账入口，允许收起。
- 智能匹配属于业务动作，不属于信息架构导航；使用顶部高对比主操作按钮“开始智能匹配”，并在政策详情中保留上下文入口。
- 业务主色：`#F8DF70`；信息蓝仅用于可点击名称和正文链接，红色不用于普通操作按钮。

## 3. 表格和卡片的使用边界

### 必须使用表格

以下场景需要横向比较，不使用卡片流：

- 居民档案列表。
- 智能匹配结果。
- 核实任务台账。
- 政策版本、办理记录和操作日志。

表格规则：

- 表头使用浅黄色背景，与全站主色保持一致。
- 普通行高控制在44–56px。
- 居民姓名、政策名称使用文字链接。
- 多个属性使用小型标签，不使用大色块。
- 居民列表固定姓名列和操作列。
- 居民主表只展示居民、村组、户籍状态、重点标签、匹配政策、更新时间和操作；家庭情况、已享受政策及待核实项放入展开档案或画像页。
- 复杂内容允许当前行展开，禁止为摘要信息频繁新增页面。
- 横向滚动时仍需保留对象名称和操作入口。

### 适合使用卡片

以下场景允许使用卡片：

- 政策库首页，政策数量较少且需要展示摘要。
- 居民画像中的单项政策权益。
- 工作台顶部的统计摘要。
- 同类居民等少量关联对象。

卡片规则：

- 圆角4–6px，不使用20px以上大圆角。
- 内边距14–18px。
- 普通卡片不设置过高的固定高度。
- 阴影仅用于建立层级，优先使用边框。
- 卡片必须包含明确标题、摘要、状态或下一步操作。
- 同一行卡片高度接近，但不以大量空白强行等高。

## 4. 信息密度

- 页面顶部说明控制在一至两行。
- 筛选区分为常用筛选和更多筛选。
- 常用筛选最多展示3个字段。
- 重点人群、匹配状态等业务分组使用页签。
- 默认最多展示3个重点标签或政策结果，超出后显示数量入口。
- 表格单元格最多展示两至三行摘要，完整内容在展开区域或抽屉查看。
- 同一屏优先呈现查询条件、结果数量和至少两条数据。

## 5. 颜色规则

| 用途 | 颜色 | 说明 |
| --- | --- | --- |
| 全站主色 | `#F8DF70` | 标题栏、主按钮、选中页签和重点边框 |
| 文字颜色 | `#111111` | 标题、正文、链接、标签、状态和按钮文字统一使用 |
| 深金色 | `#7A6200` | 仅用于边框、悬停底色和选中装饰，不用于文字 |
| 页面背景 | `#F3F5F7` | 全站背景 |
| 内容背景 | `#FFFFFF` | 表格、卡片、抽屉 |
| 成功绿 | `#16835A` | 高度匹配、已完成、已享受 |
| 待办橙 | `#C77708` | 待核实、材料缺失、待处理 |
| 异常红 | `#BD3E38` | 失败、逾期和数据异常 |
| 中性灰 | `#667382` | 辅助信息、暂不匹配 |

状态不能只依赖颜色，必须同时显示文字。

## 6. 字体和间距

标题字号参考[北京市大兴区产业发展服务门户](https://chanye.bjdx.gov.cn/chanye/zcfw37/index.html)的实际计算样式（2026-07-12核验）：门户主视觉标题为34px、字重700、行高67px；“热门政策”“政策匹配”“政策解读”等一级内容标题为24px、字重500、行高24px。

- 门户主视觉标题：34px / 700 / 67px，仅用于未来门户首页或大型品牌主视觉，业务工作台不得使用。
- 一级模块及详情实体标题：24px / 600–700 / 1.35，例如居民姓名、政策名称和抽屉主标题。
- 一级内容分区标题：24px / 500–600 / 1.35。列表页若已经通过导航和页签明确模块，不重复展示该标题。
- 卡片及二级区块标题：16px / 700 / 1.35。
- 表格表头和小型标签：12px / 700–800 / 1.35。
- 表格和正文：13–14px。
- 辅助信息：12px。
- 关键统计数字：20–26px。
- 正文、链接、标签、按钮、数据和状态文字统一使用400常规字重。
- 只有页面实体标题、一级/二级区块标题、卡片标题、表格表头和平台名称使用700粗体。
- 基础间距采用4px网格：4、8、12、16、20、24px。
- 普通控件高度：34–36px。
- 主按钮高度：36px左右。

## 7. 筛选和页签

- 姓名搜索、年龄、乡村网格属于居民列表常用筛选。
- 姓名搜索固定放在“人员档案列表 / 乡村网格视图”页签右侧。
- 重点人群、匹配状态、年龄、网格和筛选操作优先保持在同一行，控件使用紧凑宽度。
- 性别、政治面貌、工作单位等放入“更多筛选”。
- 重点人群和政策匹配状态使用页签，不使用长下拉列表。
- 已生效筛选应显示数量，清空操作始终可见。
- 返回列表时保留搜索、筛选、分页、滚动位置和展开状态。

## 8. 状态和标签

- 属性标签：高龄、独居、低收入、残疾等，使用蓝灰色。
- 成功状态：高度匹配、已办结，使用绿色。
- 待核实状态：待核实、材料收集中，使用橙色。
- 中性状态：暂不匹配、暂无登记，使用灰色。
- 标签高度约22–24px，圆角3–4px。
- 每个区域避免同时出现过多高饱和颜色。

## 9. 页面模板

### 列表页

状态页签 → 搜索与筛选 → 结果数量 → 数据表格 → 分页。不要重复显示模块名称和介绍文案。

### 详情工作台

对象摘要 → 关键统计 → 待核实事项 → 核心业务卡片 → 关联对象 → 固定免责声明。

### 任务台账

任务统计 → 状态页签 → 任务表格 → 当前行更新状态或右侧抽屉处理。

### 智能匹配

当前政策与范围 → 高度匹配/待核实/暂不匹配页签 → 结果表格 → 匹配依据 → 生成核实任务。

## 10. 可用性和安全

- 所有可点击元素必须有明显悬停和键盘焦点。
- 功能按钮必须使用填充背景或完整边框，不允许只依赖文字判断是否可点击。
- 正文中的可跳转名称和文字操作默认显示1px下划线，悬停时下划线加粗；不可跳转正文不使用下划线和悬停底色。
- 主要操作使用黄色填充按钮，次要操作使用白底描边按钮；轻量文字操作可以使用带下划线的文字按钮。
- 正文和背景保持足够对比度。
- 抽屉和弹窗支持明确关闭按钮。
- 匹配结果固定提示“系统结果仅供工作人员辅助核查，不作为正式审批结论”。
- 手机号、身份证号和详细地址默认脱敏。
- 不使用匹配百分比替代具体依据。
- 无数据、加载失败和权限不足必须提供下一步说明。
